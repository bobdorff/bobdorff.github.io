Map attack!

11/22/2015

I am not a cartographer, but today we're going to talk all about mapping. Specifically, we're going to delve into the map method that lives inside Ruby. This method is part of a larger module known as "Enumerables." A complete explination of modules and how they're used is beyond the scope of this blog post, but it's worth knowing that modules are collections methods that can be "mixed in" to your programs. Once you've mixed in a module like "Enumerable," you have access to a bunch of methods that will make your life easier. One of these incredibly useful methods is called map.

Enumerable methods are used to manipulate collections of objects. When you use the map method, you instruct the computer to work through a collection of items and perform a specific code block on each item it comes across. Then, map returns a new array that contains the results of the code.

If you've worked with arrays and hashes in Ruby before, you're probably thinking that map sounds a lot like each. These methods are similar, in that each allows you to work through a collection and modify the collection's contents with a block of code, but they return different things. When you use each on an array, the returned value is the original array. In contrast, map will return a new array that reflects the changes caused by your code block.

To drive this home, let's look at some examples. If we have the following array

array = [1, 2, 3]

map and each will give different return values.

array.each { |item| item + 1 }

=> [1, 2, 3]

while

array.map { |item| item + 1 }

=> [2, 3, 4]

It's important to recognize that the array map provides does not overwrite the original array. It's an entirely new entity. If you do want to overwrite the old array, however, Ruby will let you. With the map! method, you can modify an array's contents in place, and the original array (with newly altered values) will be returned.
